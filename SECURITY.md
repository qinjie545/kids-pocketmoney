## 报告安全漏洞

如果您发现零钱管理系统中的安全漏洞，请通过以下方式报告：

### 🔒 首选报告方式

**请勿在公开的GitHub Issues中报告安全漏洞**

发送电子邮件至：qinjie545@163.com

请在邮件中包含：

- 漏洞的详细描述
- 重现步骤
- 潜在影响评估
- 建议的修复方案（如果有）
- 您的联系信息（用于跟进）

### 其他联系方式

如果您无法发送邮件，可以通过以下方式联系：

- 创建私有安全咨询：[GitHub Security Advisories](https://github.com/qinjie545/kids-pocketmoney/security/advisories/new)
- 直接联系维护者（通过GitHub私信）qinjie545@163.com

## 响应时间

我们致力于快速响应安全报告：

- **初始响应**: 收到报告后24-48小时内
- **漏洞确认**: 3-5个工作日内
- **修复时间**: 根据漏洞严重程度而定
- **披露时间**: 修复后立即公开（或协商披露日期）

## 安全更新流程

1. **接收报告** - 安全团队评估漏洞
2. **确认漏洞** - 验证漏洞的有效性和影响
3. **开发修复** - 创建安全补丁
4. **测试修复** - 确保修复不破坏现有功能
5. **发布更新** - 推送安全补丁到受支持版本
6. **公开披露** - 在修复发布后公开漏洞详情

## 已知安全注意事项

### 数据安全

- 用户密码使用Werkzeug安全哈希存储
- 数据库文件应放置在安全位置
- 默认用户凭据仅用于演示，应立即更改

### 网络安全

- 默认运行在localhost，生产环境应配置适当访问控制
- 建议使用HTTPS进行生产部署
- API端点应验证用户身份

### 依赖安全

我们定期监控和更新第三方依赖：

- Flask 3.0.0+
- Werkzeug 3.0.1+
- APScheduler 3.10.4+

## 最佳实践

### 生产部署建议

1. **更改默认凭据**
   ```python
   # 修改app.py中的默认用户
   ```

2. **使用强密码**
   - 最少8个字符
   - 包含字母、数字和特殊字符

3. **配置访问控制**
   - 使用反向代理（如Nginx）
   - 启用HTTPS
   - 配置防火墙规则

4. **定期备份**
   - 备份数据库文件
   - 验证备份完整性

5. **监控日志**
   - 启用Flask调试日志
   - 监控异常和错误

### 开发安全

- 在开发环境中使用测试数据
- 不要在代码中硬编码敏感信息
- 定期运行安全扫描工具
- 遵循最小权限原则
